# CAT1: rollout checklist листингов

## Автоматические ворота

Перед выпуском обязательны:

- `npm run check`
- `npm test`
- `npm run build`
- `npm run test:e2e:listings` на целевом окружении

Smoke-матрица проверяет:

- базовую категорию;
- категорию с одним фильтром;
- категорию с двумя фильтрами;
- каталог брендов;
- акции;
- поиск и активный ценовой фильтр;
- сохранение исходной верхней границы цены после фильтрации;
- защиту от диапазона `250–0`;
- переход из листинга в товар;
- мобильный листинг в тёмной теме.

Workflow `.github/workflows/listing-smoke.yml` запускается вручную и после успешного production deploy.

## Ручная приёмка

### Desktop

- Карточки не двигаются и не меняют размер при hover.
- Цена, кнопки и изображения не создают layout shift.
- Полный диапазон цены не отображается как активный фильтр.
- Сброс цены возвращает обе ручки к доступному минимуму и максимуму.
- Каждый active-filter chip снимается отдельно; `Сбросить все` очищает весь набор.
- Пустая выдача объясняет причину и предлагает сброс.

### Mobile

- Фильтры открываются одной панелью и не конфликтуют с сортировкой.
- Панель прокручивается независимо, кнопки доступны при высоте 667 px.
- После применения фильтра панель закрывается, выдача остаётся на месте.
- Нет горизонтального скролла и наложения нижней навигации.

### Индексация

- Базовая категория: indexable, canonical на себя.
- Одобренный одиночный фильтр: indexable, canonical на управляемый URL.
- Неодобренный одиночный фильтр: `noindex`, canonical на категорию.
- Два и более фильтра: `noindex`, canonical на категорию.
- `sort`, `layout`, `show`, `page`: не создают самостоятельный canonical.
- `/search`: `noindex`.

## Rollback

1. Откатить последний storefront commit через обычный revert.
2. Если проблема только в управляемом листинге, перевести его в `draft`/`archive` без отката приложения.
3. Не удалять additive-таблицы и данные при rollback.
4. Повторно запустить `Listing smoke matrix` на production.
