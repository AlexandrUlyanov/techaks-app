# SEO-208 — Шаблон контентного обогащения карточки товара

## Цель

Сделать карточки товара лучше для:

- Яндекса и коммерческой индексации;
- AI-поиска и factual extraction;
- быстрого чтения человеком перед покупкой.

Важно: не добавляем фейковые преимущества, не придумываем характеристики и не пишем маркетинговую воду, если для неё нет основания в данных.

## Базовый шаблон карточки

### 1. Title

Формула:

`{название товара}, {ключевая характеристика} — купить в Пензе в ТЕХАКС`

Если товара нет в наличии:

`{название товара}, {ключевая характеристика} — цена и характеристики в ТЕХАКС`

### 2. Meta description

Составляем из четырёх честных частей:

1. название товара;
2. бренд;
3. цена;
4. факт о наличии / самовывозе / доставке.

Пример:

`Wi‑Fi роутер Hoco AX1500 RD04, бренд Hoco, цена 2 990 ₽. Кратко: Wi‑Fi роутер • AX1500 RD04 • до 1201 Мбит/с. Товар есть в наличии, доступен самовывоз в Пензе и доставка по России.`

### 3. AI-friendly factual block

Минимум фактов, которые должны быть видны сразу:

- Цена
- Бренд
- Тип
- Модель
- Цвет
- Совместимость / интерфейс / разъём
- Категория
- Артикул
- Наличие в Пензе

Если какого-то факта нет в данных, мы его не выдумываем и не показываем.

### 4. Краткий summary block

Одна короткая строка для быстрого чтения:

- Тип
- Модель
- 1 ключевая характеристика

Пример:

`Наушники • EW55 • Bluetooth 5.3`

### 5. Основные характеристики

Верхний приоритет для витрины и SEO:

- Тип
- Модель
- Цвет
- Совместимость
- Подключение / разъём
- Размер / диагональ / мощность / объём памяти — по контексту товара
- Гарантия

Если характеристик много, в верхний factual block попадают только самые полезные для выбора.

### 6. Наличие и получение

В карточке и в SSR-блоке должны быть честно отражены:

- наличие в магазинах;
- самовывоз в Пензе;
- доставка по России;
- отсутствие товара, если его нет.

### 7. Отзывы

Если отзывы есть:

- показываем 1–2 опубликованных отзыва в SSR-блоке;
- добавляем structured data review / aggregateRating.

Если отзывов нет:

- показываем честный empty-state:
  `Отзывов пока нет. Будьте первым, кто оставит отзыв о товаре после покупки — это поможет другим быстрее принять решение.`

### 8. FAQ block

Для каждой карточки нужен компактный factual FAQ:

- что важно знать о товаре перед покупкой;
- есть ли наличие в Пензе;
- можно ли забрать самовывозом;
- доступна ли доставка.

## Минимум характеристик по типам товаров

### Смартфоны / планшеты

- бренд
- модель
- цвет
- диагональ экрана
- память
- аккумулятор
- гарантия

### Наушники / аудио

- тип
- модель
- подключение
- версия Bluetooth / интерфейс
- время работы
- цвет

### Кабели / зарядки / аксессуары

- тип
- модель
- разъём
- совместимость
- мощность / ток / протокол
- длина

### Роутеры / сетевые устройства

- тип
- модель
- стандарт Wi‑Fi / интерфейс
- скорость
- диапазон
- питание / подключение

## Приоритет ручной доработки

Карточка попадает в верх приоритета, если одновременно соблюдаются 2–3 условия:

- коммерчески важная категория;
- описание короче 160 символов;
- меньше 5 полезных характеристик;
- 0 отзывов;
- товар уже видим в каталоге и имеет цену;
- есть спрос из поиска / брендовый трафик / листинг на главной.

## Готовая SQL-эвристика для shortlist SKU

Запускать на production/staging DB:

```sql
SELECT
  p.slug,
  p.name,
  COALESCE(p.article, '') AS article,
  c.name AS category_name,
  p.price,
  COALESCE(JSON_LENGTH(p.specs), 0) AS specs_count,
  CHAR_LENGTH(TRIM(REGEXP_REPLACE(COALESCE(p.description, ''), '<[^>]+>', ' '))) AS description_length,
  p.review_count
FROM products p
INNER JOIN categories c ON c.id = p.category_id
WHERE p.is_active = 1
  AND COALESCE(p.is_auto_blocked, 0) = 0
  AND p.price > 0
ORDER BY
  CASE
    WHEN LOWER(c.name) REGEXP 'смартф|планшет|станц|наушник|роутер|заряд|кабель|колонк|часы|щетк' THEN 0
    ELSE 1
  END,
  CASE
    WHEN CHAR_LENGTH(TRIM(REGEXP_REPLACE(COALESCE(p.description, ''), '<[^>]+>', ' '))) < 160 THEN 0
    ELSE 1
  END,
  CASE
    WHEN COALESCE(JSON_LENGTH(p.specs), 0) < 5 THEN 0
    ELSE 1
  END,
  p.review_count ASC,
  p.price DESC
LIMIT 50;
```

## Волна 1 для ручного enrichment

В первую очередь стоит дорабатывать SKU из групп:

- смартфоны и телефоны;
- планшеты;
- Яндекс Станции;
- роутеры и сетевые адаптеры;
- наушники;
- зарядные устройства и кабели;
- умные щётки и бытовые гаджеты, которые уже ранжируются по long-tail.
