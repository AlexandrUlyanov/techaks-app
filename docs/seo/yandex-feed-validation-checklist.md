# Yandex YML: чек-лист загрузки и валидации

Дата: 2026-06-14  
Контур: SEO-205

## Перед загрузкой в Яндекс

Открыть:

- `/admin/feeds`
- блок `Yandex YML`
- нажать `Проверить Yandex`

## Что проверяем

1. Публичный URL отдаёт XML с HTTP 200  
   `https://techaks.ru/feeds/yandex-business.yml`

2. В фиде есть офферы  
   Если офферов 0, проверяем:
   - включён ли канал;
   - есть ли видимые товары;
   - не отсекаются ли товары по цене / наличию / категориям.

3. Нет критичных пропусков:
   - нулевая цена;
   - отсутствующее изображение;
   - пустая категория;
   - пустое имя товара.

4. Желательно закрыть предупреждения по качеству:
   - нет `vendorCode`;
   - нет `barcode`;
   - пустой `description`;
   - бренд определился как `Без бренда`.

5. Проверка картинок:
   - в админке смотрим `Подозрительные картинки из выборки`;
   - если URL возвращает 403/404/timeout, картинку надо исправить до новой выгрузки.

## Правила выгрузки

- товары с нулевой ценой не должны попадать в фид;
- товары без доступного изображения не должны попадать в фид;
- товары без категории не должны попадать в фид;
- товары без названия не должны попадать в фид;
- товары без наличия выгружаются только если это явно разрешено настройкой канала.

## После загрузки в Яндекс

Проверяем в Яндекс Товарах / Бизнес:

- принят ли XML без ошибок;
- нет ли всплеска отклонённых офферов;
- совпадают ли цена и наличие с storefront;
- нет ли жалоб на товарные идентификаторы и изображения.

## Когда нужно реагировать сразу

Сразу чиним, если:

- XML перестал открываться;
- резко упало число офферов;
- появились массовые пропуски по цене / картинке / categoryId;
- выборочная проверка картинок показывает недоступные URL;
- Яндекс начал отклонять офферы, которые раньше принимал.
