# Чек-лист синхронизации с Яндекс Бизнес для Techaks

Дата: 2026-06-06  
Проект: Techaks

## 1. Базовые сущности

- Основной домен в карточке организации: `https://techaks.ru`
- Главное зеркало сайта: `https://techaks.ru`
- Регион присутствия: `Пенза`
- Основной телефон совпадает на сайте, в Яндекс Бизнес и в schema.org
- Основной e-mail совпадает на сайте, в Яндекс Бизнес и в schema.org

## 2. Адреса и точки продаж

- В Яндекс Бизнес заведены все публичные магазины Techaks
- Названия магазинов совпадают с `/stores` и `/contacts`
- Адреса магазинов совпадают с `/stores`, `/contacts` и `Store` schema
- Для каждой точки заполнены:
  - телефон
  - режим работы
  - ссылка на карту / карточку
  - категория бизнеса

## 3. Сигналы доверия на сайте

- Страница `/contacts` отдаёт 200 и содержит:
  - телефон
  - e-mail
  - адреса магазинов
  - часы работы
- Страница `/stores` отдаёт 200 и содержит:
  - список точек
  - адреса
  - часы работы
  - контакты
- В футере совпадают:
  - телефон
  - e-mail
  - краткий адрес

## 4. Schema / микроразметка

- На главной есть `Organization`
- На `/contacts` и `/stores` есть `Organization` + `Store`
- В `Organization.address` используется публичный адрес, а не случайное/устаревшее значение
- В `Store` schema совпадают:
  - `name`
  - `address`
  - `telephone`
  - `openingHours`

## 5. Региональные формулировки

- На главной есть явная привязка к Пензе
- В шаблонах категорий есть упоминание самовывоза в Пензе
- В карточках товара есть упоминание самовывоза в Пензе / доставки по России
- Коммерческие страницы не противоречат региону сайта

## 6. После обновлений

- После изменения адресов/телефонов обновить:
  - настройки сайта
  - `/contacts`
  - `/stores`
  - schema-поля
  - карточки Яндекс Бизнес
- После выкладки проверить:
  - `https://techaks.ru/contacts`
  - `https://techaks.ru/stores`
  - `https://techaks.ru/`
  - наличие актуального адреса и телефона в crawler HTML
